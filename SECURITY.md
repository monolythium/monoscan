# Security Policy

Please report vulnerabilities privately to security@monolythium.com.

Do not open a public issue for active vulnerabilities, exposed secrets, exploitable RPC behavior, wallet interaction bugs, or deployment misconfiguration that could put users or operators at risk.

Include:

- A concise description of the issue.
- Steps to reproduce, if available.
- Affected commit, deployment, or configuration.
- Any logs or payloads needed to confirm the issue, with secrets redacted.
